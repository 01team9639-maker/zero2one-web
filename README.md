# ZERO 2 ONE — Marketing Agency Website

Marketing website for **ZERO 2 ONE**, a Riyadh-based digital marketing agency.
Static, animation-driven site (no backend / build step) with a bilingual
English ⇄ Arabic layer and a small Python audit toolkit under [`tools/`](tools/).

> **Tagline:** *from zero to one* — web design, SEO, advertising, brand
> identity, social media, and e-commerce.

---

## Tech stack

| Concern            | Library / approach |
|--------------------|--------------------|
| Page transitions   | [barba.js](https://barba.js) (`sync` AJAX, swaps `<main>`) |
| Smooth scroll      | locomotive-scroll |
| Animation          | GSAP + ScrollTrigger |
| Lazy loading       | vanilla-lazyload |
| Fonts              | Neue Montreal / *Dennis Sans* (local) + **Cairo** (Google, Arabic) |
| i18n               | custom dictionary + DOM `TreeWalker` — [`assets/js/i18n.js`](assets/js/i18n.js) |
| Analytics          | Google Analytics (gtag) |

All third-party JS is loaded from public CDNs (jsDelivr / cdnjs / jQuery CDN).

## Running locally

It's fully static — serve the folder with any static server:

```bash
python3 -m http.server 8000
# then open http://localhost:8000
```

Opening `index.html` via `file://` mostly works but a server is recommended
(barba fetches pages over HTTP).

## Bilingual (English default, Arabic auto)

`assets/js/i18n.js` detects Arabic from the visitor's device
(`navigator.languages` **and** the OS locale via `Intl…resolvedOptions().locale`)
and, if Arabic:

- sets `<html lang="ar" dir="rtl">` + `body.lang-ar`,
- walks the text nodes inside `<main>` and swaps English → Arabic from a
  dictionary (the loading screen is intentionally left untranslated),
- applies a few `SPECIAL` innerHTML swaps (hero marquee, footer heading, logo
  hover label),
- CSS in the **Arabic / RTL** block of `style-new.css` handles direction fixes.

Force a language for testing with `?lang=ar` or `?lang=en` (stored in
`localStorage`).

## SEO

Every page ships with: a unique `<title>` + meta description, canonical URL,
Open Graph + Twitter cards, `theme-color`, and JSON-LD structured data.
`robots.txt` + `sitemap.xml` are present and validated.

## GEO / AEO (AI answer engines)

Beyond classic SEO, the site is tuned to be **understood and cited by
generative / answer engines** (ChatGPT, Claude, Perplexity, Gemini, Google AI
Overviews):

- **Rich structured data** — an `@graph` of `ProfessionalService`
  (with an `OfferCatalog` of all six services, full NAP, `sameAs`) and
  `WebSite` on the home page; `Service` + `BreadcrumbList` on each service page;
  and an **`FAQPage`** backed by a **visible FAQ section** (direct answers are
  what AI Overviews quote).
- **[`llms.txt`](llms.txt)** — an LLM-friendly summary of the agency, services
  and contact details (the emerging GEO standard).
- **[`robots.txt`](robots.txt)** — explicitly welcomes AI crawlers (GPTBot,
  OAI-SearchBot, ClaudeBot, PerplexityBot, Google-Extended, Applebot-Extended,
  CCBot…).

Both are verified by the toolkit:

```bash
python3 tools/audit.py          # SEO / code audit          (0 errors = clean)
python3 tools/geo_audit.py      # GEO / AEO / AIO readiness  (verdict: EXCELLENT)
python3 tools/check_links.py    # broken-link report
```

The full toolkit — audits, link check, robots validator, sitemap generator,
image/CDN prep — is documented in **[`tools/README.md`](tools/README.md)**.

### ⚠️ Before going live — configure these

| Item | Where | Note |
|------|-------|------|
| **Domain** | canonical / `og:url` / `sitemap.xml` / `robots.txt` | Currently assumes `https://zero2one.sa`. If the real domain differs, re-run `tools/generate_sitemap.py --base https://REAL_DOMAIN` and update the `<head>` URLs. |
| **Analytics** | `<head>` gtag snippet | The template's GA tag has been removed; a commented GA4 placeholder is in place — drop in your own Measurement ID (`G-XXXXXXXXXX`) and uncomment to enable. |
| **Share image** | `og:image` | Uses `assets/images/about-us-zero2one.png` (portrait). A dedicated 1200×630 landscape image is recommended. |

## Images & CDN

Source images are heavy PNGs. `tools/optimize_images.py` generated **WebP**
copies next to each one (**≈92 % smaller — 26 MB → 2.2 MB**) and losslessly
re-compressed the PNGs.

To actually serve the savings, pick one:

1. **CDN auto-format (recommended, zero markup change).** Put Cloudflare
   (enable *Polish → WebP*) or Bunny (*Optimizer*) in front of the origin. The
   CDN converts/upgrades images per request; the existing `assets/images/*.png`
   URLs keep working.
2. **`<picture>` element.** Reference the generated `.webp` with a PNG
   fallback:
   ```html
   <picture>
     <source srcset="assets/images/service-1.webp" type="image/webp">
     <img src="assets/images/service-1.png" alt="…">
   </picture>
   ```
3. **Point assets at a CDN host.** If assets live on a CDN subdomain, rewrite
   `assets/…` references to `https://cdn.zero2one.sa/assets/…` at deploy time.

Regenerate anytime: `python3 tools/optimize_images.py --png`
(add `--max-width 1920` to also cap oversized sources).

## Deployment

Any static host works (Netlify, Vercel, Cloudflare Pages, S3+CloudFront, Nginx).
Deploy the repo root as-is. Recommended headers/CDN settings:

- long-cache `assets/**` (fingerprint or cache-bust on change),
- enable Brotli/Gzip for HTML/CSS/JS,
- enable image auto-format (see above),
- serve over HTTPS (canonical URLs are `https://`).
